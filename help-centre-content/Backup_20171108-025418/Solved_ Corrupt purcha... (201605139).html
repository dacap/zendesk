Solved: Corrupt purchase order template layout in Reckon Accounts

Issue:

The layout of the Received column does not print properly in the purchase order template. We are awaiting feedback from Reckon on the actual cause, however, we have a workaround solution.

Software affected:

  • Reckon Accounts 2011, 2012, 2013 Enterprise Edition
  • Quickbooks 2011, 2012, 2013 Enterprise Edition

Example:

Notice the layout of the Rec'vd column is incorrect in this purchase order. Continue on to see how this can be resolved.

Resolution:

The following steps will resolve the print layout of purchase order.

  1. Customize purchase order

    In Reckon Accounts, open a purchase order and click Customize.

  2. Additional Customization

    Click Additional Customization in the Basic Customization window.

  3. Edit Columns

    Click the Columns tab in the Additional Customization window.

  4. Untick the problematic column

    Untick the print check box for the Received column or the column that is giving you grief.

  5. Apply changes

    Click OK to apply changes

  6. Go back into Additional Customizations

    Click Additional Customizations in the Basic Customizations window.

  7. Go back into Columns

    Click the Columns tab in the Additional Customization window.

  8. Tick the problematic column

    Tick the print checkbox for the Received column or the column that is giving you grief.

  9. Customize column order

    Enter in your preferred column order, in this example we are placing the Received column before the Back-ordered column.

  10. Review customizations

    Click Print Preview to confirm the template is now printing properly.

  11. Success!

    The layout of the Received column is now fixed.
